# 场景
比如: 
- 一个表单提交界面, 用户输入了各种信息后, 点击提交按钮, 但是因为网络延迟可能导致一定时间内没响应, 
用户不知道问题所在可能会多次点击按钮,带来了高并发, 增加了服务器的压力


# 没有意义的请求
1. 提交按钮 --- 点击后立即禁用按钮, 避免用户重复点击
2. 防抖


# 防抖
- 在规定的时间内没有新的触发, 才执行

1. 借助闭包保存上一次的定时器
2. 每次触发都清除上一次的定时器, 然后重新设置定时器
3. 考虑参数问题
4. this指向问题


# 节流
- 有节制的执行: 在一定时间内, 只执行一次
- 考虑参数问题
- this指向问题
